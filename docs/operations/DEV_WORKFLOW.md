# Development Workflow

## Branching
- Feature branches for isolated work.
- Small, reviewable commits.

## Minimum checks per change
- Lint/format (when toolchain is selected).
- Unit tests for game-core logic.
- Simulation checks for balance-sensitive mechanics.

## PR quality bar
- Clear change summary.
- Updated docs for any rule/content change.
- Notes on migration if content format changes.
